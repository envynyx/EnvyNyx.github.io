Our Commitment & Experience

Resonant Core is committed to building platforms that are efficient, secure, easy-to-use, and maintainable. We build upon the resilience of open source software and contribute our innovations back to the community.

Business Successes

  • EMR Integration

    Resonant Core employees worked with an online doctor appointment service to integrate their appointment scheduling process with several third-party EMR providers, resulting in less frustration for doctors and patients alike.

    We also identified and repaired several programming mistakes made by an offshore team that previously worked on the project that could have resulted in a full server compromise and complete access to protected health information. (Thankfully, it did not.)

  • Android App Migration

    Members of the Resonant Core team worked with one of the largest cell tower construction companies to migrate an incredibly slow Android native app to a modern, cross-platform HTML5 architecture with offline capability (acheived through use of the application cache and HTML5's localStorage feature).

    In addition to a significant speed increase from both a superior architecture and a relational database backend, the new app also offered in-app photo editing and annotations, automated reporting, and an ad-hoc reporting tool.

Improving Open Source Projects / Security Research

  • Facebook SDK v4

    We improved the random number generator used throughout the Facebook Software Development Kit to produce cryptographically secure output, which nontrivially increases the security of the functions that rely on randomness in apps that use newer versions of the Facebook SDK.

  • WordPress Core

    A member of Resonant Core enhanced WordPress's random number generator to rely on the system entropy pool instead of a userspace random number generator.

    We also audited and advocated a proposed feature that would allow WordPress's auto-update mechanism to deliver digitally signed updates (with an offline key) to protect the tens of millions of WordPress users in the event of a wordpress.org compromise.

  • CodeIgniter Framework

    One of our team members identified weaknesses with the CodeIgniter session library that, in the hands of a sophisticated attacker, could result in remote code execution. The discussion that followed resulted in a new, stronger encryption library consistent with industry best practices. The vulnerabilities we identified were fixed in CodeIgniter 2.2.0 and the relevant feature is slated to be removed in verison 3.

  • Yii Framework v2

    When reviewing the Yii2 framework, we discovered an inconsistency in how random numbers were being generated. While most of their security code was following industry best standards, their password hashing library was relying on a Merseinne Twister for random salts (which can be predictable). Predictable salts can lead to salted hash collisions (if the passwords are the same). This was fixed before Yii2 was officially released.

  • The PHP League's OAuth2 Server

    Resonant Core's consultants improved the security of OAuth2 Server library provided by the League of Extraordinary PHP Packages by ensuring the key generation algorithm always returns the correct key size.

    We also identified that their hash_equals() implementation had the side-effect of defining the function inside of their current namespace. While this is not a security or performance concern, it does violate software engineering best practices; so, we fixed that too.
